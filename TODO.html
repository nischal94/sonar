Dogfood checklist

Order matters — if step 2 fails, step 4 is a waste of Apify budget.

Swap .env placeholders for real keys
OPENAI_API_KEY, GROQ_API_KEY, APIFY_API_TOKEN, RESEND_API_KEY, RESEND_FROM_EMAIL. Leave Twilio/Telegram as placeholders — Wizard/Dashboard/Backfill don't touch them.
Run the Wizard on yourself
Enter your real capability profile. Watch what keywords the LLM proposes — too broad? too narrow? This feedback loop is more valuable than any synthetic test. Bump PROMPT_VERSION on every content change.
Install extension/ as unpacked in Chrome
Let it observe your real LinkedIn feed for a few hours. Does the content script's DOM selector survive LinkedIn's actual markup? Does capture-connections.js scroll the virtualized list correctly? These have never run against live LinkedIn.
Click "Run day-one scan"
~$1.50 of real Apify money (harvestapi at $1.50/1k posts, capped at 10 posts/profile). Does it return posts? Do they match ApifyProfilePost? If either fails, check backfill-apify-research.md §5 — the schema may have drifted and _map_row needs a field-access tweak.
Live with the dashboard for 3–5 days
Is 0.65 the right default threshold? Are matching-signal-phrase summaries useful or LLM slop? Is 30s polling noisy? Is the "Draft outreach" button begging to be wired up?

Definition of done: one GitHub issue per real bug, plus a gut-check answer to "if this were a stranger's product, would I use it for a second day?" Yes → ship Discovery. No → the issue list tells you what to fix first.
